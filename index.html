Portal Link Planner

Plan Minecraft nether portal placements that link the way you intend. Enter your portals, say which one should lead where, and the planner simulates the game's real linking rules for every trip. If something is wrong, the solver proposes coordinates that work. Everything runs in your browser.

How portal linking works

    
Portals do not remember each other. Every time you step through one, the game re-runs the same four rules. This is what the planner simulates.


    

      

        
1. The Nether is 8 times smaller

        
Walking 1 block in the Nether equals 8 blocks in the Overworld. When you enter a portal, the game divides your X and Z by 8 (or multiplies by 8 going the other way) to find where you should come out. Height (Y) is never scaled.

        
Rounding always goes down, which matters for negative coordinates: -2513 divided by 8 gives -315, not -314.

      

      

        
      

    


    

      

        
2. The game searches a square zone

        
Around that scaled arrival spot, the game looks for existing portals in a square area: 16 blocks each way in the Nether, 128 each way in the Overworld (on Java 1.16+; older versions and Bedrock use 128 for both).

        
Height does not matter for being found. A portal on the nether roof is still spotted if its X and Z are inside the square. A portal 17 blocks off sideways is invisible, no matter its height.

      

      

        
      

    


    

      

        
3. The closest portal wins, measured in 3D

        
If several portals are inside the square, the game picks the one closest to the arrival spot in full 3D, and here height suddenly does count.

        
A portal on the nether roof (around Y 128) starts with a 40-block distance handicap against ground-level portals. Experienced players use that handicap deliberately to control which portal wins.

      

      

        
      

    


    

      

        
4. Link stealing, and how to beat it

        
Two bases close together in the Overworld end up with arrival spots right next to each other in the Nether. One nether portal can then sit inside both search zones and hijack its neighbour's traffic.

        
The fix: move one portal so it is outside the other base's search zone (more than 16 blocks from that arrival spot) while staying inside its own. Each base can then only ever find its own portal. This is exactly what the solver works out for you, including the return trips, where the times-8 scaling amplifies any offset.

        
One more subtlety the planner handles for you: the game uses your position for the divide-by-8 math, and a portal is 2 blocks wide, so the arrival spot can shift by 1 block depending on where you stand. That is why margins under 3 blocks are flagged as fragile.
