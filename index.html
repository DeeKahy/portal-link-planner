Plan Minecraft nether portal placements that link the way you intend — verify every trip, catch link-stealing, and auto-solve safe coordinates. Runs entirely in your browser.

How portal linking actually works (Java 1.16+)
  1. No stored links. Portals never remember a partner — every trip recalculates the destination from scratch.
  2. Coordinate scaling. Overworld → Nether: (floor(x/8), y, floor(z/8)). Nether → Overworld: (x×8, y, z×8). Y is never scaled. Floor semantics matter for negatives: floor(−2513/8) = −315.
  3. Candidate search. The game collects every existing portal block whose X and Z are within a square (Chebyshev) around the target: ±16 in the Nether, ±128 in the Overworld (±128 both in Bedrock / pre-1.16). The full Y column is searched — Y never limits candidacy.
  4. Winner selection. Among candidates, the portal block with the smallest 3D Euclidean distance to the target wins. Y matters here. No candidates → a brand-new portal spawns (usually a planning error).
  5. Target wobble. You stand somewhere inside a 2-block-wide frame, so the computed target can shift ±1 in X or Z. Robust plans survive the worst case — this tool checks every variant.

Two independent ways portal A beats portal B: exclusion (B is outside the search square — the strongest lock, achievable in Java's ±16 Nether) and distance dominance (both are candidates, but A is 3D-closer — big Y offsets, like a roof portal at Y≈128 vs targets at Y≈87, create a built-in handicap). Always check the return trip too: a nether portal 20 blocks off-target is 160 overworld blocks off after ×8 scaling — outside the return square.
